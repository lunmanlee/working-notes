Publish pipeline — the blueprints.

What we're building: a one-command path from an Obsidian note to the live site, with created / modified dates that live in the note itself so they never drift. Each panel below maps to a step in the plan. Toggle ☾ to read them on warm paper or at night.

The note record

entity relationship

The shape build.js emits into notes.js for each note. Frontmatter created / updated source planted / tended; in-folder wikilinks form a self-referential graph that yields backlinks and the status heuristic.

step 1

build.js — frontmatter & dates

flowchart

Each note is split into frontmatter and body. Dates come from created / updated when present, falling back to filesystem stats so the build never breaks. The --- block is stripped before rendering.

flowchart TD
  A[/"note.md"/] --> B{"has frontmatter?"}
  B -->|yes| C[parseFrontmatter]
  B -->|no| D[whole file is body]
  C --> E["data: created, updated"]
  C --> F["body without ---"]
  D --> F
  E --> G{"date present?"}
  G -->|yes| H["toYMD → planted / tended"]
  G -->|no| I[("fs.statSync(file) → birthtime / mtime")]
  I --> H
  F --> J["renderMarkdown → HTML"]
  H --> K["clamp planted ≤ tended"]
  J --> L[note record]
  K --> L
  L --> M[["notes.js"]]
step 2

Backfill migration

flowchart

One-time, idempotent. Seeds the 26 existing notes with their real file dates as frontmatter so the plugin doesn't reset them to today.

flowchart TD
  S(["for each *.md in vault"]) --> H{"already has created:?"}
  H -->|yes| K[skip — leave as is]
  H -->|no| R[("fs.statSync(file) → birthtime / mtime")]
  R --> C["clamp + fmtDate"]
  C --> P["prepend created / updated block"]
  P --> W[write note back]
  K --> N([next file])
  W --> N
durability

Note lifecycle

state

Dates ride inside the note, so a clone on any machine rebuilds the same dates. Growth status is independent and displayed, not a gate.

stateDiagram-v2
  [*] --> Draft: new note
  Draft --> Stamped: plugin writes created + updated
  Stamped --> Edited: I edit it
  Edited --> Stamped: updated bumped
  Stamped --> Published: npm run publish
  Published --> Cloned: git clone elsewhere
  Cloned --> Stamped: dates intact (from frontmatter)
  Published --> [*]
step 3

npm run publish

sequence

The one command. Build, then commit + push only if something changed; Cloudflare picks up the push and deploys. Live in ~30s.

sequenceDiagram
  autonumber
  actor Me
  participant N as npm / publish.sh
  participant B as build.js
  participant V as Vault
  participant G as git → GitHub
  participant C as Cloudflare Pages
  Me->>N: npm run publish
  N->>B: node build.js
  B->>V: read *.md (+ frontmatter)
  V-->>B: notes
  B-->>N: notes.js written
  N->>N: any git changes?
  alt changes exist
    N->>G: add · commit "notes: YYYY-MM-DD" · push
    G->>C: webhook (push to main)
    C->>C: deploy static files
    C-->>Me: live at *.pages.dev (~30s)
  else nothing changed
    N-->>Me: "nothing to publish"
  end
